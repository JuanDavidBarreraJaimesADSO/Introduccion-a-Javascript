Hoisting en javascript 

Es un comportamiento de JavaScript que hace que las variables y las funciones declaradas se trasladen (hoist) al principio del ámbito en el que se declaran. Esto significa que las variables y las funciones declaradas se pueden usar antes de que se declaren realmente. 

El hoisting en JavaScript permite que las variables y funciones se utilicen antes de ser declaradas.

En este caso, la declaración de var x se "eleva" al inicio del contexto de ejecución, pero la asignación se queda en su lugar original.


Imprimirá "undefined" porque la variable x se declara antes de que se use. Sin embargo, debido al hoisting, la variable x se traslada al principio del ámbito de la función foo(), por lo que la función puede acceder a ella.


Variable

Con la palabra clave var. “Ya no es usada”  Por ejemplo, var x = 42. Esta sintaxis se puede utilizar para declarar variables locales y globales, dependiendo del contexto de ejecución.
Con la palabra clave const o let. Por ejemplo, let y = 13. Esta sintaxis se puede utilizar para declarar una variable local con ámbito de bloque.
Una variable en JavaScript es un contenedor para almacenar datos. Se puede declarar con var, let, o const.



Ámbitos

Ámbito global: El ámbito global es el ámbito más amplio en JavaScript. Todas las variables y funciones declaradas fuera de cualquier función se encuentran en el ámbito global.


Ámbito de bloque: El ámbito de bloque es un ámbito creado por las llaves {}. Las variables y las funciones declaradas dentro de un bloque se encuentran en el ámbito de bloque.


Ámbito de comportamiento: El ámbito de comportamiento es un ámbito creado por una función. Las variables y las funciones declaradas dentro de una función se encuentran en el ámbito de comportamiento de la función.

Tipos de operadores:


JavaScript tiene varios tipos de operadores, como aritméticos (+, -, *, /), de comparación (==, !=, ===, !==, <, >, <=, >=), lógicos (&&, ||, !), de asignación (=, +=, -=, *=, /=), entre otros.


Los operadores son herramientas que nos permiten hacer cálculos o comparaciones en JavaScript. Por ejemplo, podemos sumar números (+), comparar si son iguales (==), o asignar un valor a una variable (=).



1. **Operadores Aritméticos**:
   - `+` : Suma
   - `-` : Resta
   - `*` : Multiplicación
   - `/` : División
   - `%` : Módulo (resto de la división)
   - `**` : Potencia

2. **Operadores de Asignación**:
   - `=` : Asignación
   - `+=` : Suma y asignación
   - `-=` : Resta y asignación
   - `*=` : Multiplicación y asignación
   - `/=` : División y asignación
   - `%=` : Módulo y asignación

3. **Operadores de Comparación**:
   - `==` : Igual a (compara el valor)
   - `===` : Igual a y del mismo tipo (compara el valor y el tipo)
   - `!=` o `!==` : No igual a
   - `>` : Mayor que
   - `<` : Menor que
   - `>=` : Mayor o igual que
   - `<=` : Menor o igual que

   - `&&` : AND lógico
   - `||` : OR lógico
   - `!` : NOT lógico






5. **Operador Ternario (Conditional)**:

6. **Operadores de Incremento/Decremento**:
   - `++` : Incremento
   - `--` : Decremento

7. **Operadores Bit a Bit**:
   - `&` : AND a nivel de bits
   - `|` : OR a nivel de bits
   - `^` : XOR a nivel de bits
   - `~` : NOT a nivel de bits
   - `<<` : Desplazamiento a la izquierda
   - `>>` : Desplazamiento a la derecha
   - `>>>` : Desplazamiento a la derecha sin signo

8. **Operadores de Concatenación**:
   - `+` : Concatenación de cadenas

Clases  y Herencia

En JavaScript, una clase es una plantilla para crear objetos que define propiedades y métodos. Las clases se definen con la palabra clave class y el nombre de la clase.


  class Persona {
    constructor(nombre, edad) {
      this.nombre = nombre;
      this.edad = edad;
    }
 
    saludar() {
      console.log(`Hola, soy ${this.nombre}`);
    }
  }
 
  const persona1 = new Persona("Juan", 23);
  persona1.saludar(); // Salida: "Hola, soy Juan"
 


La herencia con clases en JavaScript se realiza utilizando la palabra clave “extends. “

La herencia es un mecanismo que permite a una clase heredar las propiedades y métodos de otra clase. La clase que hereda se llama clase derivada o subclase, y la clase de la que hereda se llama clase base o superclase.



  class Estudiante extends Persona {
    constructor(nombre, matricula) {
      super(nombre);
      this.matricula = matricula;
    }
 
    estudiar() {
      console.log(`Estoy estudiando.`);
    }
  }
 




Métodos estáticos  Get y Set 

Métodos estáticos:

Son funciones asociadas a una clase, pero no a un objeto en particular.
Se invocan utilizando el nombre de la clase seguido del punto y el nombre del método, sin necesidad de instanciar la clase.
Se utilizan para realizar operaciones que no dependen del estado de un objeto individual, sino de la clase en sí.


  class MathUtils {
    static factorial(n) {
      if (n === 0 || n === 1) {
        return 1;
      } else {
        return n * MathUtils.factorial(n - 1);
      }
    }
  }
 
  console.log(MathUtils.factorial(5)); // Imprime 120
 


 class MathHelper {
    static sumar(a, b) {
      return a + b;
    }
  }
 
  console.log(MathHelper.sumar(3, 4)); // Salida: 7
 









Getters:
Son funciones especiales que se definen en un objeto para controlar el acceso a una propiedad.
Se invocan como si fueran propiedades del objeto, utilizando la notación de punto (.).
Permiten realizar operaciones adicionales al momento de "leer" la propiedad, como validaciones o cálculos.
	
  class Persona {
    constructor(nombre) {
      this._nombre = nombre;
    }
 
    get nombre() {
      return this._nombre.toUpperCase();
    }
  }
 
  const persona1 = new Persona("Juan");
  console.log(persona1.nombre); // Salida: "JUAN"
 
Setters:
Son funciones especiales que se definen en un objeto para controlar la asignación a una propiedad.
Se invocan como si fueran propiedades del objeto, pero utilizando el operador de asignación (=).
Permiten realizar operaciones adicionales antes de asignar un valor a la propiedad, como validaciones o conversiones.

const persona = {
    _edad: 0,
    set edad(nuevaEdad) {
      if (nuevaEdad >= 0) {
        this._edad = nuevaEdad;
      } else {
        console.error("La edad no puede ser negativa");
      }
    },
  };
 
  persona.edad = 25;
  persona.edad = -10; // Imprime un error en la consola
 
  console.log(persona.edad); // Imprime 25

  class Persona {
    constructor(nombre) {
      this._nombre = nombre;
    }
 
    get nombre() {
      return this._nombre;
    }
 
    set nombre(nuevoNombre) {
      this._nombre = nuevoNombre.trim();
    }
  }
 
  const persona1 = new Persona("   Juan   ");
  console.log(persona1.nombre); // Salida: "Juan"
 
  persona1.nombre = "  María  ";
  console.log(persona1.nombre); // Salida: "María"
 


Objeto console 


El objeto console en JavaScript proporciona acceso a la consola de depuración del navegador o del entorno de ejecución actual. Esta consola permite emitir mensajes, realizar inspecciones de variables, ejecutar código y mucho más.
Métodos principales del objeto console:
console.log(...): Imprime un mensaje o una lista de argumentos separados por espacios en la consola.
console.info(...): Similar a console.log, pero suele utilizarse para mensajes informativos menos críticos.
console.warn(...): Imprime un mensaje de advertencia en la consola, que suele estar coloreado de amarillo para llamar la atención.
console.error(...): Imprime un mensaje de error en la consola, que suele estar coloreado de rojo para indicar un problema.
console.debug(...): Similar a console.log, pero su uso suele estar habilitado solo en entornos de desarrollo para mensajes de depuración más específicos.

Operador de corto circuito javascript 


Los operadores de cortocircuito en JavaScript son dos:
Operador AND lógico (&&)
Operador OR lógico (||)
Estos operadores se llaman "de cortocircuito" porque detienen la evaluación de una expresión tan pronto como se conoce el resultado final.


Operador AND lógico (&&)
El operador && se evalúa de izquierda a derecha. Si el primer operando es false, no se evalúa el segundo operando y se devuelve false. Si el primer operando es true, se evalúa el segundo operando y se devuelve el valor del segundo operando.

const a = 10;
const b = 0;
const resultado = a > 0 && b > 0;
console.log(resultado); // Imprime "false" (b es menor que 0)


Operador OR lógico (||)
El operador || se evalúa de izquierda a derecha. Si el primer operando es true, no se evalúa el segundo operando y se devuelve true. Si el primer operando es false, se evalúa el segundo operando y se devuelve el valor del segundo operando.

const a = 10;
const b = 0;


const resultado = a > 0 || b > 0;


console.log(resultado); // Imprime "false" (b es menor que 0)

  console.log("El usuario canceló la acción.");

Expresiones regulares (regex)

Las expresiones regulares (regex) son patrones que se utilizan para buscar y modificar cadenas de texto. Son una herramienta poderosa para procesar texto en JavaScript y en muchos otros lenguajes de programación.

Componentes básicos de las regex:
Caracteres literales: Coinciden con sí mismos.
Metacaracteres: Tienen un significado especial en las regex.
Construcciones: Combinan caracteres y metacaracteres para crear patrones más complejos.
^[a-z]+$: Coincide con cualquier cadena que solo contenga letras minúsculas.
\d{3}-\d{3}-\d{4}: Coincide con un número de teléfono de EE. UU.
.*@.*\.com: Coincide con una dirección de correo electrónico.
Se pueden usar con las funciones String.prototype.match(), String.prototype.search(), String.prototype.replace(), y String.prototype.split().
